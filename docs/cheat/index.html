Your rules

  1. Every /command = you type it. Never auto-run.
  2. After /wake: plan shown, waits for go.
  3. Newborns RO: /spawn → /wake → wait.
  4. PR branches for multi-file changes. g.js routing = always PR.
  5. ~80+ calls = degraded. /spawn fresh.
  6. Writes additive. No exclusive access.
  7. Files >4KB: you push (anti-exfil protocol).

Spawn mechanics

  • agentId="bsLmGiLFdV" required, plain @sharpy = noop
  • Workers fully isolated, inline everything
  • No callbacks, poll via search_conversations
  • 3 project_conversation servers = failure domains

Anti-exfil protocol

  • ≤4KB: agent pushes via single GET cb=
  • >4KB: user handoff (gh CLI or GitHub UI)
  • Upload slots deprecated for AI use
  • Never chunk, never multi-step encode
  • CORS + POST planned (#54)

Watch for

  • /transcend aborts if any convo has pending @sharpy
  • Parallel spawns: /wake only, no /sync
  • ~5KB raw = gateway URL limit; bigger = user handoff
  • Closing </textarea> in file content breaks encoder
  • Underscore prefix on api/ subdirs = Vercel ignores (12-fn limit)
  • Batch commit chokes on multi-byte UTF-8; use a=u instead
